feat: Implementação completa do sistema RBAC (Role-Based Access Control)

BREAKING CHANGE: Estrutura de roles migrada de enum hard-coded para sistema
dinâmico baseado em banco de dados com permissões granulares.

## 🎯 Resumo

Implementado sistema completo de RBAC que permite gestão dinâmica de roles
e permissões via interface administrativa, eliminando necessidade de code
changes para criar novos roles.

## ✨ Principais Funcionalidades

- **Gestão dinâmica de roles**: MASTER pode criar roles customizados via UI
- **Permissões granulares**: 30 permissões organizadas em 7 categorias
- **Hierarquia de níveis**: Sistema de níveis 0-100 para controle fino
- **Cache otimizado**: Cache in-memory com TTL de 5 min para performance
- **Interface administrativa**: UI completa para gestão de roles e permissões
- **Backward compatibility**: Mantém campo legacyRole durante transição

## 🗄️ Database

### Novos Modelos

- `Role`: Roles do sistema (USER, SUPPORT, GERENTE, ADMIN, MASTER) + customizados
- `Permission`: Permissões granulares (30 permissões em 7 categorias)
- `RolePermission`: Tabela de associação many-to-many

### Modificações

- `User.roleId`: Foreign key para Role (novo)
- `User.legacyRole`: String role (mantido para compatibilidade)

### Seeds Executados

- `rbac-seed.ts`: 5 roles + 30 permissões + 83 associações
- `migrate-users-to-rbac.ts`: 5 usuários migrados

## 🚀 APIs Implementadas

```
GET    /api/v1/roles                      - Lista roles
GET    /api/v1/roles/:id                  - Busca role específico
POST   /api/v1/roles                      - Cria role customizado
PUT    /api/v1/roles/:id                  - Atualiza role
DELETE /api/v1/roles/:id                  - Deleta role customizado
GET    /api/v1/roles/permissions/all      - Lista permissões
POST   /api/v1/roles/:id/permissions      - Adiciona permissão
DELETE /api/v1/roles/:id/permissions/:pid - Remove permissão
PUT    /api/v1/roles/:id/permissions      - Substitui permissões
```

## 🛡️ Segurança

- Todas as rotas protegidas por `masterMiddleware` (MASTER only)
- Rate limiting: 20 operações/hora
- Validação de hierarquia em mudanças de role
- Cache de permissões com TTL de 5 minutos
- Audit log de todas as operações
