## 🎯 Objetivo da Sessão

Implementar módulo de simulação de depósito de colateral para facilitar testes do sistema de saldo interno, eliminando a necessidade de fazer transações reais na blockchain durante o desenvolvimento.

---

## 📋 Contexto

Na sessão anterior (23/10/2025), foi implementado o **Sistema de Saldo Interno (v0.3.0)**, que permite usuários depositarem colateral uma vez e criarem múltiplos pedidos instantâneos.

**Problema identificado:**
- Para testar o sistema, era necessário fazer depósitos reais de criptomoeda
- Testes lentos (10-30 minutos para confirmação na blockchain)
- Custos de taxas de rede para cada teste
- Dificuldade de testar fluxos completos rapidamente

**Solução proposta:**
Criar endpoint e interface de simulação que permita "simular" o recebimento de depósitos instantaneamente, sem transações reais.

### 1. Backend: Endpoint de Simulação

**Arquivo**: `apps/api/src/controllers/collateral-balance.controller.ts`

#### Novo Método: `simulateDeposit()`

```typescript
async simulateDeposit(req: Request, res: Response) {
  // SECURITY: Bloquear em produção
  if (process.env.NODE_ENV === 'production') {
    return res.status(403).json({
      success: false,
      message: 'Endpoint de simulação não disponível em produção',
    });
  }

  const userId = req.user?.userId;
  const { addressId } = req.params;

  // Validações:
  // 1. Usuário autenticado
  // 2. Endereço existe
  // 3. Endereço pertence ao usuário
  // 4. Status é AWAITING_PAYMENT

  // Simular recebimento do depósito
  const simulatedTxHash = `0xSIMULATED${Date.now()}${Math.random().toString(36).substring(7)}`;

  // Atualizar CollateralAddress
  await prisma.collateralAddress.update({
    where: { id: addressId },
    data: {
      status: 'CONFIRMED',
      txHash: simulatedTxHash,
      actualAmount: collateralAddress.expectedAmount,
      confirmedAt: new Date(),
    },
  });

  // Creditar saldo interno
  await internalBalanceService.creditDeposit(
    userId,
    collateralAddress.cryptoType,
    collateralAddress.cryptoNetwork,
    actualAmount,
    simulatedTxHash
  );

  // Retornar sucesso
}
```

**Características:**
- ✅ Proteção de segurança: bloqueado em produção
- ✅ Validação completa de usuário e endereço
- ✅ TxHash simulado realista: `0xSIMULATED{timestamp}{random}`
- ✅ Atualiza status do endereço para `CONFIRMED`
- ✅ Credita saldo interno usando serviço existente
- ✅ Registra transação de auditoria automaticamente
- ✅ Logs detalhados para debugging

#### Nova Função: `handleSimulateDeposit()`

```typescript
const handleSimulateDeposit = async () => {
  if (!depositAddress?.id) {
    alert('Nenhum endereço de depósito encontrado');
    return;
  }

  if (!confirm('🧪 Simular depósito?\n\nIsso vai creditar o saldo instantaneamente (APENAS TESTE)')) {
    return;
  }

  setSimulatingDeposit(true);

  try {
    const token = localStorage.getItem('accessToken');

    const response = await fetch(
      `http://localhost:3001/api/v1/collateral-balance/simulate-deposit/${depositAddress.id}`,
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${token}`,
        },
      }
    );

    const data = await response.json();

    if (data.success) {
      alert(`✅ Depósito simulado com sucesso!\n\n${data.data.amount} ${data.data.cryptoType} creditado no seu saldo interno.`);

      // Fechar modal e atualizar dados
      setShowAddModal(false);
      setDepositAddress(null);
      setDepositAmount('');
      setSelectedCrypto('');
      setSelectedNetwork('');

      // Recarregar saldos e transações
      await fetchBalances();
      await fetchTransactions();
    } else {
      alert(data.message || 'Erro ao simular depósito');
    }
  } catch (error) {
    console.error('Erro ao simular depósito:', error);
    alert('Erro ao simular depósito');
  } finally {
    setSimulatingDeposit(false);
  }
};
```

**Características:**
- ✅ Confirmação antes de simular
- ✅ Loading state durante processamento
- ✅ Feedback de sucesso detalhado
- ✅ Auto-fechamento do modal
- ✅ Auto-refresh de dados (saldos + transações)
- ✅ Tratamento de erros

**Linhas adicionadas**: +51

## 🚀 Como Usar

### Passo a Passo

1. **Iniciar aplicação**:
   ```bash
   # API
   cd apps/api
   npm run dev

   # Web
   cd apps/web
   npm run dev
   ```

2. **Acessar página de saldo**:
   - URL: `http://localhost:3000/collateral-balance`

3. **Adicionar colateral**:
   - Clique em "➕ Adicionar Colateral"
   - Selecione crypto (ex: BTC)
   - Selecione rede (ex: BITCOIN)
   - Digite valor (ex: 0.001)
   - Clique em "Gerar Endereço de Depósito"

4. **Simular depósito**:
   - Aparecerá endereço gerado
   - **Clique no botão roxo** "🧪 Simular Depósito (Teste)"
   - Confirme no dialog
   - Aguarde processamento (<1s)
   - ✅ Saldo creditado!

5. **Verificar resultado**:
   - Veja novo saldo na lista
   - Confira transação no histórico
   - Tipo: `DEPOSIT`
   - TxHash: `0xSIMULATED...`
