### Tarefa #19: Correção de Constraint Unique
**Problema**: Erro `Unique constraint failed on the fields: (address)` ao criar múltiplos pedidos.

**Causa**: Campo `address` em `CollateralAddress` tinha constraint `@unique`, mas o sistema usa o mesmo endereço da plataforma para múltiplos depósitos.

**Solução**:
- Removida constraint `@unique` do campo `address`
- Adicionado índice `@@index([address])` para performance
- Aplicado com `prisma db push`

**Arquivo modificado**:
- `/apps/api/prisma/schema.prisma`

### Tarefa #22: Correção Final do Resumo Financeiro (Colateral NÃO é Devolvido)
**Problema**: Usuário identificou informações incorretas e enganosas.

**Erros encontrados**:
1. Informava que "O colateral será devolvido quando a transação for concluída" (FALSO)
2. Taxa mostrada como 1.5% ao invés de 2.5%
3. Não explicava que o colateral vai para o pagador

**Fluxo correto**:
- PIX de R$ 10.000
- Criador deposita R$ 10.250 em BTC (valor + 2.5%)
- Pagador paga R$ 10.000 via PIX
- Pagador recebe R$ 10.100 em BTC (valor + 1% cashback)
- Plataforma fica com R$ 150 em BTC (1.5% fee)
- Criador recebe R$ 10.000 em BRL
- **Colateral NÃO volta para o criador - vai para o pagador**

**Correção implementada**:
```
Sobre o Colateral:
- Valor depositado: 0.01526196 BTC
- Taxa total (2.5%): -0.00038155 BTC
  • 1.5% vai para a plataforma
  • 1% vai como cashback para quem pagar

⚠️ O colateral NÃO será devolvido
Ele será transferido para quem pagar seu PIX.
Você receberá os R$ 10.000,00 em BRL.
```

**Mudanças**:
- ✅ Taxa corrigida: 2.5% total (1.5% plataforma + 1% cashback)
- ✅ Aviso em destaque amarelo: "O colateral NÃO será devolvido"
- ✅ Explicação clara: Colateral vai para pagador, criador recebe BRL
- ✅ Removida informação enganosa sobre devolução

**Arquivo modificado**:
- `/apps/web/app/orders/[orderId]/page.tsx` (linhas 498-523)

## 🧪 Como Testar o Sistema Completo

### 1. Cadastrar Endereço da Plataforma
```bash
# Login como MASTER
1. Acesse: http://localhost:3000/login
2. Login: master@mktplace.com / Master@2025!
3. Clique em "Meu Perfil" → Redireciona para /admin
4. Clique em "Endereços da Plataforma"
5. Adicione endereço BTC/BITCOIN (ou qualquer cripto)
```

### 2. Criar Pedido como Usuário Normal
```bash
# Login como usuário comum
1. Crie uma conta ou use uma existente
2. Acesse: http://localhost:3000/orders/create
3. Preencha formulário (PIX ou Boleto)
4. Escolha cripto que você cadastrou
5. Clique "Depositar Colateral em Cripto"
```

### 3. Simular Pagamento de Colateral
```bash
# Na tela de QR Code
1. Veja o QR Code e timer de 30 minutos
2. Clique no botão verde "⚡ SIMULAR PAGAMENTO (TESTE)"
3. Sistema marca colateral como confirmado
4. Pedido é criado automaticamente
5. Redireciona para "Meus Pedidos"
```

### 4. Verificar no Marketplace
```bash
# Como outro usuário
1. Acesse: http://localhost:3000/marketplace
2. Veja o pedido que você acabou de criar
3. Pedido aparece com colateral confirmado ✅
```

### 5. Filtrar Pedidos
```bash
# Em "Meus Pedidos"
1. Clique em diferentes filtros
2. Veja contadores atualizando
3. Teste filtro "Cancelados"
```

### 6. Ver Resumo Financeiro Correto
```bash
# Abrir detalhes do pedido
1. Clique em um pedido
2. Veja resumo financeiro corrigido
3. Informações claras sobre o que receberá
```
