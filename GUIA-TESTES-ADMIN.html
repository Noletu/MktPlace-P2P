Guia de Teste — Mktplace P2P

Perspectiva do Administrador de Negócio
Atualizado em: 22/03/2026

Para quem é este guia: Alguém sem conhecimento técnico que quer verificar se a plataforma funciona corretamente do ponto de vista do negócio.
Siga os cenários em ordem — cada um constrói sobre o anterior.

Pré-requisito: Ter o sistema rodando.
  • Frontend: http://localhost:3000
  • Backend: http://localhost:3001

Antes de Começar — Criar Contas de Teste

Você vai precisar de 3 contas para testar o ciclo completo. Crie todas em /register antes de iniciar:

ContaEmail sugeridoPapel
Adminadmin@teste.comAdministrador da plataforma
Vendedorvendedor@teste.comUsuário que vende crypto
Compradorcomprador@teste.comUsuário que compra crypto

MÓDULO 1 — Acesso ao Painel Administrativo

Teste 1.1 — Login como Administrador

  1. Acesse http://localhost:3000/login
  2. Entre com a conta de admin
  3. ✅ Esperado: Redireciona para o dashboard do usuário comum
  4. Na barra de endereço, vá para http://localhost:3000/admin
  5. ✅ Esperado: Painel admin carrega com menu de navegação completo
  6. ❌ Falha se: Aparecer "Acesso negado" ou redirecionar para login
Se aparecer "Acesso negado", a conta admin precisa ter nível ≥ 40. Peça ao desenvolvedor para promover a conta.

Teste 1.2 — Navegação do Menu Admin

Clique em cada item do menu e marque os que carregam corretamente:

  • Dashboard (/admin) — gráficos e estatísticas
  • Usuários (/admin/users) — lista de usuários
  • Pedidos (/admin/orders) — lista de pedidos
  • Audit Log (/admin/audit) — histórico de ações
  • Marketplace (/admin/marketplace) — pedidos disponíveis
  • Suporte (/admin/support) — tickets de suporte
  • Disputas (/admin/disputes) — disputas entre usuários
  • Cupons (/admin/coupons) — gestão de cupons
  • Segurança (/admin/security) — 2FA do admin
  • Roles (/admin/roles) — níveis de acesso

✅ Todos devem carregar sem erro.

MÓDULO 2 — Gestão de Usuários

Teste 2.1 — Visualizar Usuários

  1. Acesse /admin/users
  2. Você deve ver a lista com os 3 usuários criados
  3. ✅ Esperado: Tabela mostra nome, email, reputação, limite diário
  4. Teste o filtro: digite o email do vendedor na busca
  5. ✅ Esperado: Lista filtra mostrando apenas o vendedor

Teste 2.2 — Ver Detalhes de um Usuário

  1. Na lista de usuários, clique no ícone 👁 ao lado do vendedor
  2. ✅ Esperado: Modal abre mostrando:
    • Dados pessoais (nome, email, data de cadastro)
    • Score de reputação atual
    • Limite diário de transação
    • Histórico de pedidos e carteiras

Teste 2.3 — Ajustar Limite Diário de um Usuário

  1. Na lista de usuários, clique no ícone ✏ (Ajustar Limite) ao lado do vendedor
  2. ✅ Esperado: Modal abre com campo para inserir novo limite
  3. Digite 5000 (R$ 5.000) e clique em confirmar
  4. ✅ Esperado: Mensagem de sucesso; limite atualizado na lista
  5. Verifique no detalhe do usuário: deve mostrar R$ 5.000

Teste 2.4 — Congelar uma Conta

  1. Na lista de usuários, clique no ícone 🔒 (Congelar) ao lado do comprador
  2. ✅ Esperado: Modal pede motivo e data opcional de descongelamento
  3. Preencha o motivo: "Teste de congelamento" e confirme
  4. ✅ Esperado: Conta marcada como congelada na lista
  5. Abra aba anônima, logue como comprador e tente criar um pedido
  6. ✅ Esperado: Banner de "conta congelada" aparece e impede a ação

Teste 2.5 — Descongelar a Conta

  1. No admin, localize o comprador congelado e clique em 🔓 (Descongelar)
  2. ✅ Esperado: Confirmação e conta volta ao normal

Teste 2.6 — Mudar o Role de um Usuário

⚠️ Cuidado: não promova as contas de teste a MASTER.
  1. Na lista, clique em 🔄 (Mudar Role) ao lado do vendedor
  2. Selecione SUPPORT
  3. ✅ Esperado: Usuário aparece com badge de SUPPORT
  4. Reverta para USER logo em seguida

MÓDULO 3 — Criação e Ciclo Completo de Pedido

Este é o coração do negócio. Vamos simular uma venda de crypto completa.
📋 Pré-requisito: O vendedor precisa ter saldo em carteira. Em ambiente de teste, use o botão "Simular Depósito" quando disponível.

Teste 3.1 — Criar um Pedido de Venda (pelo Vendedor)

  1. Faça login como vendedor em aba anônima
  2. Acesse /orders/create e selecione:
    • Tipo: SELL (vender crypto para receber BRL)
    • Crypto: Bitcoin (ou qualquer disponível)
    • Quantidade: 0.001 BTC
    • Método de pagamento: PIX
  3. Clique em "Criar Pedido"
  4. ✅ Esperado: Pedido criado com status ACTIVE, aparece no marketplace

Teste 3.2 — Ver o Pedido no Admin

  1. No admin, acesse /admin/orders
  2. ✅ Esperado: O pedido do vendedor aparece na lista
  3. Clique em 👁 e confirme que todos os dados estão visíveis

Teste 3.3 — Aceitar o Pedido (pelo Comprador)

  1. Faça login como comprador em outra aba
  2. Acesse /marketplace, localize o pedido e clique em "Aceitar"
  3. ✅ Esperado: Pedido entra em negociação ativa

Teste 3.4 — Confirmar Pagamento e Completar

  1. Como comprador: clique em "Confirmar Pagamento Enviado"
  2. Como vendedor: verifique a notificação e clique em "Confirmar Recebimento"
  3. ✅ Esperado: Pedido muda para COMPLETED
  4. ✅ Esperado: Reputação de ambos aumenta +10 pontos

Teste 3.5 — Verificar no Admin

  1. Em /admin/orders, pedido deve aparecer como COMPLETED
  2. Em /admin/users, reputação de ambos deve estar maior
  3. ✅ Verificação de negócio: Transação registrada corretamente

MÓDULO 4 — Gestão de Disputas

Teste 4.1 — Criar uma Disputa

  1. Crie um novo pedido (repita Testes 3.1 e 3.3)
  2. Desta vez, como comprador, não confirme o pagamento
  3. Acesse /orders/[id]/dispute/new e preencha:
    • Título: "Teste de disputa"
    • Motivo: "Pagamento não confirmado"
  4. ✅ Esperado: Disputa aberta com status OPEN

Teste 4.2 — Visualizar Disputas no Admin

  1. Acesse /admin/disputes
  2. ✅ Esperado: Disputa aparece com status OPEN
  3. Veja as estatísticas no topo (total, abertas, em revisão)

Teste 4.3 — Resolver uma Disputa

  1. Clique na disputa para ver os detalhes
  2. Mude o status para "UNDER_REVIEW" (Em Análise)
  3. Resolva a favor do comprador ou vendedor
  4. ✅ Esperado: Disputa marcada como RESOLVED no audit log

MÓDULO 5 — Suporte ao Cliente

Teste 5.1 — Criar Ticket de Suporte (como Usuário)

  1. Logado como comprador, acesse /support/ticket/new
  2. Preencha: Assunto "Dúvida sobre saque" e Mensagem "Como faço para sacar meu saldo?"
  3. ✅ Esperado: Ticket criado com status OPEN

Teste 5.2 — Responder ao Ticket (como Admin)

  1. No admin, acesse /admin/support e abra o ticket do comprador
  2. Escreva uma resposta e mude o status para "RESOLVED"
  3. ✅ Esperado: Ticket marcado como resolvido

Teste 5.3 — Filtros de Suporte

  1. Filtre por status OPEN → mostra apenas abertos
  2. Filtre por prioridade HIGH → mostra apenas críticos
  3. ✅ Todos os filtros devem funcionar

MÓDULO 6 — Gestão de Cupons

Teste 6.1 — Criar um Cupom

  1. Acesse /admin/coupons e clique em "Criar Cupom"
  2. Preencha: Código TESTE10, Desconto 10%, Máx 1 uso por usuário, Público, Ativo
  3. ✅ Esperado: Cupom aparece na lista

Teste 6.2 — Usar o Cupom (como Usuário)

  1. Como comprador, crie um pedido e insira o código TESTE10
  2. ✅ Esperado: Desconto aplicado

Teste 6.3 — Verificar Uso no Admin

  1. Volte ao admin em /admin/coupons
  2. ✅ Esperado: O cupom mostra 1 uso registrado

Teste 6.4 — Desativar Cupom

  1. Mude o status do cupom para Inativo
  2. Tente usar como usuário
  3. ✅ Esperado: Sistema recusa com mensagem de erro

MÓDULO 7 — Audit Log (Rastreabilidade)

Teste 7.1 — Verificar Ações no Log

  1. Acesse /admin/audit
  2. ✅ Esperado: Todas as ações aparecem: congelamento, mudança de role, ajuste de limite, resolução de disputa
  3. Cada entrada mostra: quem fez, o que fez, quando, e se teve sucesso

Teste 7.2 — Filtrar o Log

  1. Filtre por ação "FREEZE_ACCOUNT" → só o congelamento aparece
  2. Filtre por data de hoje → só as ações de hoje aparecem

Teste 7.3 — Exportar para CSV

  1. Clique em "Exportar CSV"
  2. ✅ Esperado: Arquivo baixado com as ações registradas

MÓDULO 8 — Dashboard e Métricas

Teste 8.1 — Verificar o Dashboard Admin

  1. Acesse /admin
  2. ✅ Esperado: Números refletem os testes feitos (usuários, pedidos, volume)
  3. Gráficos devem exibir dados mesmo que pequenos

Teste 8.2 — Workers / Processos do Sistema

  1. Acesse /admin/workers
  2. ✅ Esperado: Deposit Monitor e Collateral Release aparecem como ATIVOS
  3. ❌ Atenção se: Algum worker aparecer como PARADO ou com erro

MÓDULO 9 — Robustez e Casos Extremos

Teste 9.1 — Tentar Ações Não Autorizadas

  1. Logado como vendedor (não admin), acesse http://localhost:3000/admin
  2. ✅ Esperado: Redireciona para login ou exibe "Acesso negado"

Teste 9.2 — Conta Congelada não Pode Transacionar

  1. Congele o comprador e tente fazer login e aceitar um pedido como ele
  2. ✅ Esperado: Banner de conta congelada impede a ação

Teste 9.3 — Pedido Expirado

  1. Crie um pedido com expiração mínima e aguarde expirar
  2. ✅ Esperado: Pedido muda para EXPIRED automaticamente

Teste 9.4 — Limite Diário

  1. Com conta nova (reputação 0), limite é R$ 1.000
  2. Tente criar pedido de R$ 2.000
  3. ✅ Esperado: Sistema bloqueia com mensagem de limite excedido

Teste 9.5 — Senha Errada

  1. Tente senha errada 5 vezes no login
  2. ✅ Esperado: Bloqueio temporário ou mensagem de muitas tentativas

MÓDULO 10 — Segurança do Admin (2FA)

Teste 10.1 — Configurar 2FA para o Admin

  1. Acesse /admin/security e clique em "Ativar 2FA"
  2. Escaneie o QR code com Google Authenticator, Authy ou similar
  3. Digite o código de 6 dígitos gerado
  4. ✅ Esperado: 2FA ativado, backup codes gerados
  5. Importante: Salve os backup codes em local seguro

Teste 10.2 — Login com 2FA

  1. Faça logout e logue novamente como admin
  2. ✅ Esperado: Após email/senha corretos, pede código do autenticador
  3. Digite o código do app → login bem-sucedido

Checklist Final

Fluxo de Usuários

  • Cadastro de novo usuário funciona
  • Login/logout funciona
  • Congelamento e descongelamento funciona
  • Mudança de role funciona
  • Ajuste de limite funciona

Fluxo de Negócio

  • Criar pedido de venda funciona
  • Pedido aparece no marketplace
  • Comprador consegue aceitar pedido
  • Ciclo de pagamento completa corretamente
  • Reputação aumenta após transação completada

Gestão e Controle

  • Disputas são criadas e resolvidas
  • Tickets de suporte são respondidos
  • Cupons são criados e aplicados
  • Audit log registra todas as ações admin
  • Dashboard mostra métricas corretas
  • Workers aparecem como ativos

Segurança

  • Usuário comum não acessa admin
  • Conta congelada não pode operar
  • Limite diário é respeitado
  • 2FA funciona no admin
